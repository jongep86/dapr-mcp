# Plan: Add `get_bulk_state` tool to `pkg/state`

## Context

`pkg/secrets` exposes both a single-key (`get_secret`) and a bulk (`get_bulk_secrets`)
read. `pkg/state` only exposes single-key operations (`get_state`, `save_state`,
`delete_state`, `execute_transaction`) — there is no bulk read, even though Dapr's
state API supports one (`GET /v1.0/state/<store>/bulk`, or the Go SDK's
`GetBulkState`). This adds a `get_bulk_state` tool so an agent can fetch many keys in
one call, mirroring the single/bulk pattern already established in `pkg/secrets`.

Good news from exploration: the shared mock **already declares `GetBulkState`**
(`test/mocks/dapr_client_mock.go:214`) with signature
`GetBulkState(ctx, storeName string, keys []string, meta map[string]string, parallelism int32) ([]*client.BulkStateItem, error)`,
so no mock changes are needed.

## Changes

### 1. `pkg/state/tool.go`

**a. Extend the `StateClient` interface** (line 17) with the SDK method:
```go
GetBulkState(ctx context.Context, storeName string, keys []string, meta map[string]string, parallelism int32) ([]*dapr.BulkStateItem, error)
```

**b. Add an args struct** next to the existing arg structs:
```go
type GetBulkStateArgs struct {
    StoreName   string   `json:"storeName" jsonschema:"The name of the Dapr state store component (e.g., 'statestore')."`
    Keys        []string `json:"keys" jsonschema:"The list of keys whose values should be retrieved."`
    Parallelism int32    `json:"parallelism" jsonschema:"Optional. Max number of parallel reads the state store performs; 0 lets Dapr choose a default."`
}
```

**c. Add `getBulkStateTool`** handler, following `getStateTool` and `getBulkSecretTool`:
- Open OTEL span `get_bulk_state`; set attributes `dapr.operation`, `dapr.store`,
  and `dapr.keys_count` (`attribute.Int`).
- Call `stateClient.GetBulkState(ctx, args.StoreName, args.Keys, nil, args.Parallelism)`.
- On error, return `CallToolResult{IsError: true}` with a text message (same shape as
  `getStateTool`: `"dapr GetBulkState failed: %v"`).
- On success, build the structured result mirroring what `get_state` returns per key —
  a slice of `map[string]string` with `key`, `value`, and `etag` (BulkStateItem exposes
  `Key`, `Value []byte`, `Etag`, `Error`). If an item has a non-empty `Error`, surface it
  in that item's map (e.g. `"error"` field) rather than failing the whole call.
- Return a human-readable summary text (count retrieved + store name) plus the
  structured slice as the second return value.

**d. Register the tool** in `RegisterTools` (after `get_state`, before `delete_state`):
```go
mcp.AddTool(server, &mcp.Tool{
    Name:  "get_bulk_state",
    Title: "Retrieve Multiple Keys' State in Bulk",
    Description: "Retrieves the values for multiple keys from a Dapr state store in a single call. **This is a Data Retrieval operation and IS IDEMPOTENT.** ...", // follow get_state / get_bulk_secrets wording + ARGUMENT RULES block
    Annotations: &mcp.ToolAnnotations{
        ReadOnlyHint:   isReadOnly,
        IdempotentHint: isIdempotent,
    },
}, getBulkStateTool)
```
(Reuse existing `isReadOnly`/`isIdempotent` locals — same annotations as `get_state`.)

### 2. `pkg/state/tool_test.go`

Add `TestGetBulkStateTool` following the exact table-driven style of `TestGetStateTool`,
with cases covering:
- **success** — mock returns a slice of `*client.BulkStateItem` (with Key/Value/Etag);
  assert `IsError == false` and the summary text.
- **item with per-key error** — one BulkStateItem has a non-empty `Error`; assert the
  call still succeeds and the error is surfaced in output.
- **failure** — mock returns `(nil, errors.New("connection refused"))`; assert
  `IsError == true` and content contains `"dapr GetBulkState failed"`.

Mock wiring mirrors the existing tests:
```go
m.On("GetBulkState", mock.Anything, "statestore", []string{"k1","k2"}, mock.Anything, int32(0)).
    Return([]*client.BulkStateItem{{Key:"k1", Value:[]byte(`v1`), Etag:"1"}}, nil)
```

Add a row under the `state` group (after `get_state`, line 76):
